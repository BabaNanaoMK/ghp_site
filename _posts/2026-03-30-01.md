# Claude Cowork で間取りプランナー作成

Claude Cowork で作成したプログラム

## 成果物

[間取りプランナー](https://babananaomk.github.io/room-planner/)

---

## 概要

部屋の横幅・奥行きをcm単位で指定し、家具や窓・ドアをCanvas上に配置できるWebアプリ。
サーバー不要の完全クライアントサイドアプリ（単一HTMLファイル）。
保存はブラウザのlocalStorageを使用し、JSONファイルのエクスポート/インポートにも対応。

---

## 使用技術

| 技術 | 用途 |
|---|---|
| HTML5 Canvas API | 部屋・家具・壁要素の描画 |
| Vanilla JavaScript (ES6+) | 全アプリロジック（フレームワークなし） |
| CSS Flexbox / Grid | サイドパネル・カードレイアウト |
| localStorage | 自動保存・名前付きスロット保存 |
| File API (Blob / FileReader) | JSONエクスポート・インポート |

### 外部依存なし

ライブラリ・フレームワーク・CDNは一切使用していない。`index.html` 1ファイルで完結。

## 状態管理

アプリの状態はグローバル変数で管理する。

```js
let room = { w: 400, h: 300 }        // 部屋サイズ（cm）
let items = []                         // 家具リスト
let wallElements = []                  // 窓・ドアリスト
let zoom = 1.0                         // 現在のズーム倍率
let drag = null                        // ドラッグ中の情報
let selectedColor = '#1abc9c'          // 選択中の家具色
let selectedWallType = 'window'        // 選択中の壁要素種別
```

### データ構造

**家具アイテム**
```js
{
  id: 1234567890,      // Date.now() で生成するユニークID
  name: 'シングルベッド',
  w: 97,               // 横幅（cm）
  h: 195,              // 奥行き（cm）
  x: 0,                // 配置位置・左端からのcm
  y: 0,                // 配置位置・上端からのcm
  color: '#1abc9c'
}
```

**壁要素（窓・ドア）**
```js
{
  id: 1234567890,
  type: 'window',      // 'window' | 'door'
  wall: 'top',         // 'top' | 'right' | 'bottom' | 'left'
  pos: 100,            // 壁の左端（または上端）からのcm
  width: 90            // 開口部の幅（cm）
}
```

## レンダリング

毎フレーム `render()` がキャンバス全体を再描画する（差分更新なし）。

```
render()
├── 背景・床・グリッドを描画
├── 目盛り（横・縦）を描画
├── drawWalls()                    # 壁（窓・ドア付き）
│   ├── drawWallLine()             # 壁の線（ギャップを除いたセグメントを描く）
│   ├── drawWindowSymbol()         # 青い太線でガラスを表現
│   └── drawDoorSymbol()           # 蝶番位置から扉パネル線 + 開き弧
└── items.forEach(drawItem())      # 家具（影・塗り・枠・テキスト）
```

### 座標変換

1グリッド = 10cm = 20px（zoom=1.0 のとき）。

```js
cmToPx(cm) = cm / 10 * 20 * zoom
pxToCm(px) = px / (20 * zoom) * 10
```

キャンバスの描画原点はPADDING（40px）オフセット（目盛り表示のため）。

### ドア記号の描画

蝶番を開口部の左端（または上端）に固定し、室内方向へ扉パネル（直線）と開き弧（quarter-circle）を描く。

| 壁 | 蝶番位置 | 扉パネル方向 | 弧の角度範囲 |
|---|---|---|---|
| top | 開口部左端 | 下方向（室内へ） | 0 〜 π/2 |
| bottom | 開口部左端 | 上方向（室内へ） | -π/2 〜 0 |
| left | 開口部上端 | 右方向（室内へ） | 0 〜 π/2 |
| right | 開口部上端 | 左方向（室内へ） | π/2 〜 π |

---

## ズーム

- `resetZoom()`: ウィンドウサイズと部屋サイズから最適ズームを自動計算（0.3〜1.5倍）
- `changeZoom(delta)`: ±0.1 刻みで手動調整（0.2〜3.0倍）
- ズーム変更のたびに `resizeCanvas()` でCanvasの物理サイズを再計算

## 保存・読み込み

### localStorageのキー

| キー | 内容 |
|---|---|
| `roomPlanner_autosave` | 自動保存（最新の状態、1件） |
| `roomPlanner_saves` | 名前付き保存スロット（最大10件、JSON配列） |

### 保存データのスキーマ

```js
// 状態オブジェクト（getState() の戻り値）
{
  room: { w: 400, h: 300 },
  items: [ ...家具アイテム配列... ],
  wallElements: [ ...壁要素配列... ]
}

// 名前付きスロット
{
  id: 1234567890,
  name: 'リビング案A',
  date: '2026/3/30 16:42:26',
  state: { ...状態オブジェクト... }
}
```

### 自動保存のタイミング

- 部屋サイズ変更
- 家具の追加・削除・回転・ドラッグ移動完了
- 窓・ドアの追加・削除
- 名前付きスロットからの読み込み

## 操作一覧

| 操作 | 方法 |
|---|---|
| 家具を移動 | ドラッグ（10cmスナップ） |
| 家具を回転 | ダブルクリック または サイドパネルの ↻ ボタン |
| 家具を削除 | 右クリック または サイドパネルの ✕ ボタン |
| 家具を前面へ | ドラッグ開始時に自動的に最前面へ移動 |

## デプロイ

### ローカル確認

```bash
# Python 3（任意のHTTPサーバーで可）
cd room-planner
python3 -m http.server 3000
# → http://localhost:3000
```

> **注意**: GitHub Pages で公開するには Public リポジトリが必要（無料プランの場合）。
