# シャドーイング動画検索ツール - 技術ノウハウまとめ

Claude で初プログラム作成しました。
つるつるできたのだけど、それを見ながら、

何をやってるか分からなくてもものができるのは怖い！下手なAPIをバンバン叩かれたら敵わんな

と感じました。

作るのは簡単、確認が大変

の世界が来るのでしょうね。

## 概要

多読的シャドーイング学習のために、未習得言語のYouTube動画を検索するWebツール。
日本語キーワードを入力するだけで、選択した国の言語に翻訳し、YouTube動画を検索できる。

---

## 成果物

[シャドーイング動画検索ツール](https://shadowing-video-search.onrender.com/)

## 使用技術・ライブラリ

| 技術 | 用途 | 選定理由 |
|------|------|----------|
| **Python 3** | バックエンド言語 | ライブラリが豊富、AI開発と相性が良い |
| **FastAPI** | Webフレームワーク | 軽量・高速、APIとHTMLの両方を提供可能 |
| **uvicorn** | ASGIサーバー | FastAPIの標準的なサーバー |
| **Jinja2** | HTMLテンプレート | FastAPIに統合済み、サーバーサイドレンダリング |
| **deep-translator** | 翻訳 | Google翻訳を無料・APIキー不要で利用可能 |
| **youtube-search-python** | YouTube検索 | APIキー不要でYouTube検索が可能 |
| **requests** | HTTP通信 | YouTubeサジェストAPI呼び出しに使用 |
| **Tailwind CSS なし / 素のCSS** | スタイリング | 依存を最小限にするため |

### 当初検討したが不採用にしたもの

| 技術 | 不採用理由 |
|------|-----------|
| **pytrends（Google Trends）** | ローカルでは動作するが、クラウド（Render等）からのアクセスがGoogleにブロックされる |
| **YouTube Data API v3** | APIキーが必要、無料枠に制限あり |
| **React** | 今回の規模では過剰。HTML + vanilla JSで十分 |

## アーキテクチャ

### 全体構成

```
ブラウザ (HTML + CSS + JS)
    ↕ HTTP (JSON)
FastAPI サーバー (Python)
    ├── 翻訳サービス (deep-translator → Google翻訳)
    ├── サジェストサービス (YouTube Suggest API)
    └── YouTube検索サービス (youtube-search-python)
```

**ポイント**: フロントエンドとバックエンドが1つのPythonプロセスで動作する。
`python main.py` の1コマンドで起動できるシンプル構成。

## 内部の動き（データフロー）

### ユーザー操作と処理の流れ

```
[ユーザー] 国を選択 + 日本語キーワード入力 + 「翻訳」ボタン
    │
    ▼
[POST /api/translate]
    │  ① countries.pyから国の言語コードを取得
    │  ② deep-translatorで日本語→対象言語に翻訳
    │  ③ 翻訳結果を返す
    ▼
[ユーザー] 2つのボタンが表示される
    │
    ├─→「○○で検索」ボタン ─→ [POST /api/youtube/search]
    │                              │ youtube-search-pythonで検索
    │                              ▼ 動画一覧を返す
    │
    └─→「サジェストから選ぶ」ボタン ─→ [POST /api/suggestions]
            │                              │ YouTube Suggest APIで候補取得
            │                              ▼ キーワード一覧を返す
            │
            └─→ キーワードをクリック ─→ [POST /api/youtube/search]
                                           │ youtube-search-pythonで検索
                                           ▼ 動画一覧を返す
```

### APIエンドポイント一覧

| メソッド | パス | 機能 |
|---------|------|------|
| GET | `/` | メインページ表示 |
| GET | `/api/countries` | 対応国・言語一覧（101カ国） |
| POST | `/api/translate` | 日本語キーワードを対象言語に翻訳 |
| POST | `/api/suggestions` | YouTubeサジェストキーワード取得 |
| POST | `/api/translate-to-ja` | 外国語キーワードを日本語に逆翻訳（ツールチップ用） |
| POST | `/api/youtube/search` | YouTube動画検索 |

## 各サービスの詳細

### 翻訳サービス（translator.py）

- **deep-translator** ライブラリの `GoogleTranslator` を使用
- Google翻訳のWeb版をスクレイピング（APIキー不要）
- ソース言語とターゲット言語を指定して翻訳
- 翻訳失敗時は元のキーワードをそのまま返す（エラーで止まらない）

### サジェストサービス（trends.py）

- YouTubeの検索サジェストAPI（`suggestqueries-clients6.youtube.com`）を使用
- レスポンスはJSONP形式 → パースしてJSON抽出
- 言語コード（`hl`パラメータ）で対象言語のサジェストを取得
- クラウド環境からもアクセス可能（ブロックされない）

### YouTube検索サービス（youtube.py）

- **youtube-search-python** ライブラリを使用
- キーワードで動画を検索、サムネイル・タイトル・チャンネル等を取得
- **注意**: httpxのバージョンを0.27.2に固定する必要あり（新しいバージョンだと互換性問題）

---

## 国・言語マッピング（countries.py）

- 101カ国に対応
- 各国に以下のデータを持つ：
  - `name_ja`: 日本語名（UI表示用）
  - `name_en`: 英語名（UI表示用）
  - `geo`: Google Trends用の地域コード（現在は未使用だが将来用に残存）
  - `lang`: deep-translator / YouTubeサジェスト用の言語コード

---

## フロントエンド（app.js）

- **vanilla JavaScript**（フレームワーク不使用）
- Fetch APIでバックエンドと通信
- 動的にDOM要素を生成（キーワードタグ、動画カード）
- ツールチップ: マウスホバー時に `/api/translate-to-ja` を呼んで日本語翻訳を表示
- レスポンシブデザイン対応（CSS @media クエリ）

- **Render**（https://render.com）の無料プランを使用
- GitHubリポジトリと連携、pushで自動デプロイ
- `render.yaml` にビルド・起動コマンドを定義
- 注意: Google Trendsはクラウド環境で動作しないため、YouTubeサジェストのみ使用

## 開発で得た知見・ハマりポイント

### 1. youtube-search-python と httpx の互換性問題
- `youtube-search-python` が内部で使う `httpx` の新しいバージョン（0.28以降）で `proxies` パラメータが廃止された
- **対策**: `requirements.txt` で `httpx==0.27.2` に固定

### 2. PyTrends（Google Trends）のクラウド環境での制限
- ローカルPCからは正常動作するが、Render等のクラウドサーバーからはGoogleにブロックされる
- **対策**: YouTubeサジェストAPIにフォールバック → 最終的にPyTrends自体を廃止

### 3. Python仮想環境（venv）の管理
- macOS（Homebrew管理下）ではシステムPythonに直接pipできない
- **対策**: プロジェクトごとに `python3 -m venv venv` で仮想環境を作成

### 4. deep-translatorの翻訳精度
- 単語レベルの翻訳は概ね正確だが、文脈によっては不適切な翻訳になることがある
- 例：「ドラマ」→ イタリア語「dramma」（演劇の意味になる）
- **対策**: ユーザーが結果を見て判断、サジェスト機能で代替キーワードも提供

### 5. フォルダ構成とGitリポジトリの単位
- 複数プロジェクトを管理する場合、親フォルダではなくプロジェクト単位でGitリポジトリを作る
- 各プロジェクトに独自の `venv/` を持たせる（依存ライブラリの衝突を防ぐ）
