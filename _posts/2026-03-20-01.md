# 字幕なし多観サポート - 技術ノウハウまとめ

Claudeで作成した二番目のプログラム。


## 概要

字幕なし多観学習のために、ユーザーの趣味・関心に合ったYouTube動画やチャンネルを検索するWebツール。
日本語キーワードを入力するだけで対象言語に翻訳し、動画・チャンネルを検索できる。
動画の長さ・視聴回数・いいね数・音声言語などの条件で絞り込み・並び替えが可能。

## 使用技術・ライブラリ

| 技術 | 用途 | 選定理由 |
|------|------|----------|
| **Python 3** | バックエンド言語 | ライブラリが豊富、AI開発と相性が良い |
| **FastAPI** | Webフレームワーク | 軽量・高速、APIとHTMLの両方を提供可能 |
| **uvicorn** | ASGIサーバー | FastAPIの標準的なサーバー |
| **Jinja2** | HTMLテンプレート | FastAPIに統合済み、サーバーサイドレンダリング |
| **deep-translator** | 翻訳 | Google翻訳を無料・APIキー不要で利用可能 |
| **google-api-python-client** | YouTube Data API v3 | Made for Kids・いいね数・登録者数など詳細情報の取得に必要 |
| **素のCSS** | スタイリング | 依存を最小限にするため |

### 当初検討したが不採用にしたもの

| 技術 | 不採用理由 |
|------|-----------|
| **youtube-search-python** | Made for Kidsやいいね数が取得できない |
| **React / Vue** | 今回の規模では過剰。HTML + vanilla JSで十分 |

## アーキテクチャ

### 全体構成

```
ブラウザ (HTML + CSS + JS)
    ↕ HTTP (JSON)
FastAPI サーバー (Python)
    ├── 翻訳サービス (deep-translator → Google翻訳)
    ├── YouTube動画検索サービス (YouTube Data API v3)
    └── YouTubeチャンネル検索サービス (YouTube Data API v3)
```

**ポイント**: フロントエンドとバックエンドが1つのPythonプロセスで動作する。
`python main.py` の1コマンドで起動できるシンプル構成。DBは使用しない。

## 内部の動き（データフロー）

### ユーザー操作と処理の流れ

```
[ユーザー] 検索モード・言語・入力方式・キーワードを入力
    │
    ▼（日本語入力モードの場合）
[POST /api/translate]
    │  ① languages.pyから言語コードを検証
    │  ② deep-translatorで日本語→対象言語に翻訳
    │  ③ 翻訳結果を返す
    ▼
[動画検索モードの場合]
[POST /api/youtube/videos]
    │  ① search.list（type=video）で動画ID一覧を取得（100 units）
    │  ② videos.list で詳細情報を取得（1 unit）
    │     → duration, madeForKids, viewCount, likeCount, defaultAudioLanguage
    │  ③ 長さ区分を内部計算（5分以下/10分以下/20分以下/20分超）
    │  ④ 動画一覧を返す
    ▼
[チャンネル検索モードの場合]
[POST /api/youtube/channels]
    │  ① search.list（type=channel）でチャンネルID一覧を取得（100 units）
    │  ② channels.list で詳細情報を取得（1 unit）
    │     → subscriberCount, videoCount, description
    │  ③ チャンネル一覧を返す
    ▼
[ブラウザ側] 絞り込み・並び替えはJavaScriptで処理（APIを再呼び出しせずに実行）
```

### APIエンドポイント一覧

| メソッド | パス | 機能 |
|---------|------|------|
| GET | `/` | メインページ表示 |
| GET | `/api/languages` | 対応言語一覧（133言語） |
| POST | `/api/translate` | 日本語キーワードを対象言語に翻訳 |
| POST | `/api/youtube/videos` | YouTube動画検索（詳細情報付き） |
| POST | `/api/youtube/channels` | YouTubeチャンネル検索（詳細情報付き） |

## YouTube Data API のクォータ管理

| API呼び出し | コスト | 用途 |
|---|---|---|
| `search.list` | 100 units | 動画/チャンネル検索 |
| `videos.list` | 1 unit | 動画詳細取得 |
| `channels.list` | 1 unit | チャンネル詳細取得 |
| **1回の検索合計** | **101 units** | |

- 無料枠: **10,000 units / 日**
- 1日あたり約 **99回の検索**が可能
- 上限超過時は403エラー（課金なし・翌日リセット）
- APIキーは環境変数 `YOUTUBE_API_KEY` で管理

## 各サービスの詳細

### 翻訳サービス（translator.py）

- **deep-translator** ライブラリの `GoogleTranslator` を使用
- Google翻訳のWeb版をスクレイピング（APIキー不要）
- ソース言語とターゲット言語を指定して翻訳
- 翻訳失敗時は元のキーワードをそのまま返す（エラーで止まらない）

### YouTube検索サービス（youtube.py）

**動画検索（search_videos）**
- `search.list`（type=video, relevanceLanguage指定）で動画IDを取得
- `videos.list` で `snippet,contentDetails,statistics,status` を取得
- ISO 8601 duration（PT1H2M3S）を秒数に変換して長さ区分を判定
- `defaultAudioLanguage` を取得して音声言語バッジに表示

**チャンネル検索（search_channels）**
- `search.list`（type=channel）でチャンネルIDを取得
- `channels.list` で `snippet,statistics` を取得
- 説明文は120文字で切り詰めて表示

---

## 言語マッピング（languages.py）

- Google翻訳（deep-translator）が対応する133言語をハードコード
- 各言語に以下のデータを持つ：
  - `code`: deep-translator / YouTube API用の言語コード（例: `"en"`, `"fr"`, `"zh-CN"`）
  - `name_en`: 英語名
  - `name_ja`: 日本語名（UI表示・五十音順ソートに使用）
- UI上では英語を先頭に固定し、残りを五十音順で表示

---

## フロントエンド（app.js）

- **vanilla JavaScript**（フレームワーク不使用）
- Fetch APIでバックエンドと通信
- 絞り込み・並び替えは取得済みデータをJavaScriptで処理（API再呼び出しなし）
- デフォルトソート: 動画→視聴回数順、チャンネル→登録者数順
- レスポンシブデザイン対応（CSS @media クエリ）

## デプロイ

### ローカル実行

```bash
cd jimakunashi-takan-video-search
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
export YOUTUBE_API_KEY=your_api_key_here
python main.py
# → http://localhost:8000
```

### Render（クラウド）

- **Render**（https://render.com）の無料プランを使用
- GitHubリポジトリと連携、pushで自動デプロイ
- `render.yaml` にビルド・起動コマンドを定義
- 環境変数 `YOUTUBE_API_KEY` をRenderのダッシュボードで設定
- 注意: 15分間アクセスがないとスリープし、再起動に約1分かかる

## 開発で得た知見・ハマりポイント

### 1. youtube-search-python ではなく YouTube Data API を採用
- Made for Kids（`status.madeForKids`）・いいね数（`statistics.likeCount`）は公式APIでないと取得不可
- APIキーが必要だが無料枠（10,000 units/日）で個人利用には十分

### 2. Made for Kids の登録率が低い
- 投稿者の自己申告制のため、Kids判定されている動画が少ない
- フィルタとして実用性が低いため、UIからKidsフィルタを削除

### 3. defaultAudioLanguage の空欄が多い
- 投稿者が音声言語を設定していない動画が多い
- 「対象言語のみ」フィルタでは該当なしになりやすいため「不明を含む」選択肢も提供

### 4. relevanceLanguage は完全なフィルタではない
- `search.list` の `relevanceLanguage` パラメータは関連度の優先度調整であり、他言語の動画も返る
- 厳密な言語フィルタには `defaultAudioLanguage` の照合が必要

### 5. Python仮想環境（venv）の管理
- macOS（Homebrew管理下）ではシステムPythonに直接pipできない
- プロジェクトごとに `python3 -m venv venv` で仮想環境を作成
